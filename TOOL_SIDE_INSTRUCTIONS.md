# ツール側への指示事項（Admin側の変更に伴う）

作成日: 2026-02-07

## 概要

Admin側で「基本方針・長期目標」セクションを「SNS活用の目的（Why SNS?）」に変更し、課題セクションを削除しました。ツール側（会員サイト側）の`/onboarding`ページでも同様の変更が必要です。

---

## 変更内容

### 1. セクション名の変更

**変更前:**
- 「基本方針・長期目標」

**変更後:**
- 「SNS活用の目的（Why SNS?）」

**理由:**
- 「Why SNS?」という言葉で、SNSを活用する根本的な目的が明確になる
- 計画ページの「今月の目標」と区別しやすくなる

---

### 2. 説明文の改善

**変更前:**
```
この目標は、あなたのビジネスの根幹となる基本方針です。
月ごとに変わる目標は、運用計画ページで設定できます。
```

**変更後:**
```
この目的は、あなたのビジネスがSNSで何を実現したいかを表す基本方針です。
数ヶ月〜数年変わらない、ビジネスの根幹となる目的を設定してください。
月ごとに変わる具体的な目標は、運用計画ページ（/instagram/plan）で設定できます。
```

**改善点:**
- 「数ヶ月〜数年変わらない」という期間を明記
- 「SNSで何を実現したいか」という目的を明確化

---

### 3. 課題セクションの削除

**削除理由:**
- 課題は月ごとに変わるため、月次計画ページ（`/instagram/plan`）で設定する方が適切
- 「SNS活用の目的」のみにすることで、シンプルで分かりやすくなる

**影響:**
- UIから課題入力欄を削除
- データ構造上は`businessInfo.challenges`は残す（既存データとの互換性のため）
- 月次計画ページで課題を設定できるようにする

---

### 4. プレースホルダーの改善

**変更前:**
```
例: ブランド認知度の向上
例: 顧客との関係構築
例: 新規顧客の獲得
```

**変更後:**
```
例: ブランド認知度の向上
例: 顧客との長期的な関係構築
例: 業界での専門性・信頼性の確立
例: 地域No.1の認知度獲得

※ 月ごとに変わる具体的な目標（例: 「今月は新商品の認知度向上」）は運用計画ページで設定してください
```

**改善点:**
- より具体的な例を追加
- 月次目標との違いを明確化（悪い例を提示）

---

## データ構造への影響

### 保存されるデータ（変更なし）

**`userProfile.businessInfo`:**
- `goals`: SNS活用の目的（配列）✅ 引き続き保存
- `challenges`: 課題（配列）⚠️ UIからは削除、データ構造は残す（既存データとの互換性）

**重要:**
- `challenges`フィールドはデータ構造上は残す（既存データとの互換性のため）
- UIからは削除し、月次計画ページで設定できるようにする

---

## 月次計画ページとの関係

### 役割の明確化

**`/onboarding`（SNS活用の目的）:**
- 数ヶ月〜数年変わらない、ビジネスの根幹となる目的
- 変更頻度: 低い（数ヶ月〜数年単位）
- 例: 「ブランド認知度の向上」「顧客との長期的な関係構築」

**`/instagram/plan`（今月の目標・課題）:**
- 月ごとに変わる具体的な目標と課題
- 変更頻度: 高い（月ごとに変更可能）
- 例: 「今月は新商品の認知度向上に重点を置く」「投稿頻度を週4回に増やす」

### データの優先順位

AI生成時の優先順位：
1. **最優先**: `/instagram/plan`で設定した`monthlyGoals`と`monthlyChallenges`
2. **フォールバック**: `/onboarding`の`goals`（`challenges`は削除されたため、月次計画ページのみ）

## ユーザーへの影響

### 既存ユーザー

- **影響なし**: 既存のデータはそのまま有効
- **表示の変化**: `/onboarding`ページを開くと、セクション名が変更され、課題セクションが表示されなくなります
- **既存の課題データ**: `businessInfo.challenges`はデータ構造上は残りますが、UIからは編集できなくなります

### 新規ユーザー

- **目的の明確化**: 「SNS活用の目的」として、長期にわたって変わらない目的を設定することを理解しやすくなります
- **シンプル化**: 課題セクションが削除され、設定が簡単になります
- **計画ページとの関係性**: 月ごとに変わる目標や課題は運用計画ページで設定できることを理解しやすくなります
